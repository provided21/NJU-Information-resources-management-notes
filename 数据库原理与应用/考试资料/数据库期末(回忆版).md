## 一、选择题（20分）

### 1. **ADD CONSTRAINT限制**
**题干**：可以用ADD CONSTRAINT加元组限制吗？
**答案**：❌ 错误
**考点**：约束是表级/列级规则，不能限制单个元组
**解析**：约束应用于整表或整列，限制特定元组需用触发器

## 二、填空题（10分）

### 1. **多粒度封锁优势**
**题干**：具有______的多粒度封锁方法可以降低上锁开销
**答案**：意向锁（或意向锁协议）
**考点**：意向锁减少锁检查次数，提高并发效率

### 2. **模式分解理想情况**
**题干**：模式分解的理想情况是既______又______
**答案**：既保持无损连接性，又保持函数依赖
**易错**：只写一个或顺序颠倒

### 3. **关系代数计算**
**题干**：现有两关系S(a,b,c,d)和T(c,d,e,f)，N1、N2为元组数且N2>N1。问πc,d(S)×T最多有______个元组，最少有______个元组，结果是______列。
**答案**：
- 最多：N1×N2 个元组
- 最少：N2 个元组  
- 列数：6 列
**解析**：投影去重后元组数m≤N1，笛卡尔积元组数=m×N2

### 4. **存取控制机制**
**题干**：数据库安全中，______和______是两种主要存取控制方法
**答案**：强制存取控制（MAC）、自主存取控制（DAC）
**考点**：DAC基于用户权限，MAC基于数据安全级别

### 5. **触发器挖空**
**题干**：
```sql
CREATE TRIGGER check_data
______ INSERT ON employees  -- 空1
FOR EACH ROW
BEGIN
    DECLARE new_salary INT;
    SELECT salary INTO new_salary FROM ______;  -- 空2
    
    IF new_salary < 0 THEN
        SIGNAL SQLSTATE '45000';
    END IF;
END;
```
**答案**：空1：BEFORE，空2：INSERTED
**考点**：BEFORE触发器用于数据验证，INSERTED虚拟表存新数据（题目应该不一样，但是填是让填这两个，我为了图方便给ai说了填这两个，让ai编的题目，大差不差，反正我没填对/(ㄒoㄒ)/~~）

## 三、判断题（10分）

### 1. **多值依赖性质**
**题干**：多值依赖与属性域无关，只要子集满足，整个属性集就满足
**答案**：✅ 正确
**解析**：MVD具有投影保持性，与属性域无关

### 2. **事务隔离级别**
**题干**：事务隔离等级越高，并发程度越小，数据一致性越好
**答案**：✅ 正确
**解析**：隔离级别提高 → 并发性降低 → 一致性增强

## 四、简答题（30分，6分×5）

### 1. **三级模式及优点**
**标准答案**：
- 三级模式：外模式（用户视图）、概念模式（全局逻辑）、内模式（物理存储）
- 优点：保证逻辑独立性（概念模式改，外模式可不变）和物理独立性（内模式改，概念模式可不变）

### 2. **事后安全检查操作**
**标准答案**：
- 操作：**数据库审计**（Audit）
- 原因：
  1. 安全监控与入侵检测
  2. 责任追溯与问责
  3. 威慑作用
  4. 合规性要求

### 3. **冲突可串行化判断**
**解题步骤**：
1. 找出所有冲突操作（R-W、W-W、W-R）
2. 建立前驱关系（如有冲突T1→T2）
3. 画前驱图，检查是否有环
4. 有环 → 不可冲突可串行化；无环 → 可冲突可串行化

### 4. **数据库重组与重构**
**标准答案**：
- **重组**：物理优化，不改变逻辑结构（如碎片整理、索引重建）
- **重构**：改变逻辑/物理结构（如增删表、改字段）
- **原因**：提高性能、适应需求变化、优化设计

### 5. **检查点恢复**
**标准答案**：
- **为什么用检查点**：减少恢复时间、缩小日志扫描范围、优化恢复效率
- **恢复步骤**：
  1. 找到最近一致性检查点
  2. 确定检查点时刻活跃事务
  3. 向后扫描日志，划分Redo/Undo集合
  4. Undo未完成事务（逆序）
  5. Redo已提交事务（顺序）

## 五、综合题（30分）

### **第一部分：ER图设计题（15分）**
**题目**：设计科研管理系统（项目、科研人员、单位、经费、成果）
**解题要点**：
1. **ER图**：5个实体，联系包括：
   - 项目-人员：多对多（参与，带角色属性）
   - 项目-单位：多对多（依托）
   - 项目-经费：1对多
   - 项目-成果：1对多
2. **转换关系模式**：
   - 每个实体转为关系
   - 多对多联系单独建表
   - 1对多在"多"方加外键
3. **约束**：主码、外码、非空、唯一

### **第二部分：SQL编程题（15分）**
**数据库**：SPJ（供应商-零件-工程）
1. **创建SPJ表**
   ```sql
   CREATE TABLE SPJ(
       Sno CHAR(4),
       Pno CHAR(4), 
       Jno CHAR(4),
       Qty INT,
       PRIMARY KEY(Sno, Pno, Jno),
       FOREIGN KEY(Sno) REFERENCES S(Sno),
       FOREIGN KEY(Pno) REFERENCES P(Pno),
       FOREIGN KEY(Jno) REFERENCES J(Jno)
   );
   ```

2. **查询使用S1所有零件的工程**（难点！）
   ```sql
   -- 需要除法操作
   SELECT Jno FROM J
   WHERE NOT EXISTS (
       SELECT * FROM P
       WHERE EXISTS (SELECT * FROM SPJ WHERE Sno='S1' AND Pno=P.Pno)
       AND NOT EXISTS (
           SELECT * FROM SPJ 
           WHERE Jno=J.Jno AND Pno=P.Pno AND Sno='S1'
       )
   );
   ```

3. **更新供应关系**
   ```sql
   UPDATE SPJ
   SET Sno='S3'
   WHERE Sno='S5' AND Pno='P6' AND Jno='J2';
   ```

4. **删除P6记录**（注意外键约束）
   ```sql
   DELETE FROM SPJ WHERE Pno='P6';
   DELETE FROM P WHERE Pno='P6';
   ```

5. **创建视图及查询**
   ```sql
   CREATE VIEW 一汽 AS
   SELECT Sno, Pno, Qty FROM SPJ;
   
   -- 查询零件及总量
   SELECT Pno, SUM(Qty) FROM 一汽 GROUP BY Pno;
   
   -- 查询S1参与情况
   SELECT * FROM 一汽 WHERE Sno='S1';
   ```

## 六、高频考点速记

### 🎯 **核心知识点**
1. **事务ACID**：原子性、一致性、隔离性、持久性
2. **隔离级别**：读未提交、读已提交、可重复读、序列化
3. **并发控制**：锁协议、意向锁、多粒度封锁
4. **恢复技术**：Redo/Undo、检查点、日志
5. **模式设计**：范式、函数依赖、无损分解
6. **SQL编程**：约束、视图、触发器、复杂查询
7. **ER设计**：实体联系转换关系模式

### ⚠️ **易错点提醒**
- 约束不能限制单个元组 → 用触发器
- 读已提交不能避免丢失修改 → 可重复读可以
- 多值依赖具有投影保持性 → 函数依赖不一定
- BEFORE触发器用于数据验证 → AFTER用于审计
- 1对多联系不在"多"方加外键 → 单独建表错误

### 📝 **答题技巧**
1. **简答题**：分点作答，举例说明
2. **SQL题**：先写框架，再填细节
3. **设计题**：先画ER图，再转关系模式
4. **判断题**：注意绝对化表述（"都"、"一定"）
5. **填空题**：用教材原话，写标准术语

2. 熟练SQL编程，特别是复杂查询
